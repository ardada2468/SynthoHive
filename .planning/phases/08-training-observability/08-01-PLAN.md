   ```python
   import sys
   ```

2. Change `CTGAN.fit()` signature to add two new parameters with defaults:
   ```python

           log_metrics=True, seed=None, progress_bar=True, checkpoint_interval=10, **kwargs):
   ```

3. Before the epoch loop (`for epoch in range(self.epochs):`), add:
   ```python
   # Emit training_start event
   log.info(
       "training_start",
       total_epochs=self.epochs,
       batch_size=self.batch_size,
       embedding_dim=self.embedding_dim,
       checkpoint_interval=checkpoint_interval,
   )
   _start_time = time.time()

   # Replace bare for-loop with trange (disable=True suppresses bar; log events always fire)
   pbar = trange(
       self.epochs,
       desc="Training",
       file=sys.stderr,
       leave=True,
       disable=not progress_bar,
   )
   ```

4. Replace `for epoch in range(self.epochs):` with `for epoch in pbar:`.

5. Remove the existing `if epoch % 10 == 0: print(...)` line inside the loop (replaced by tqdm postfix).

6. After the existing `# --- Checkpointing & Logging ---` block (after `current_loss_d = loss_D.item()`), add ETA calculation, tqdm postfix update, and structlog event:
   ```python
   # ETA calculation (linear extrapolation)
   _elapsed = time.time() - _start_time
   _epochs_done = epoch + 1
   _elapsed_per_epoch = _elapsed / _epochs_done
   _remaining_epochs = self.epochs - _epochs_done
   eta_seconds = _elapsed_per_epoch * _remaining_epochs  # 0.0 on final epoch — correct

   # Update tqdm postfix (visual only; fires regardless of disable state)
   pbar.set_postfix({
       "g_loss": f"{current_loss_g:.4f}",
       "d_loss": f"{current_loss_d:.4f}",
       "eta": f"{int(eta_seconds)}s",
   })

   # Emit epoch_end event (always, independent of progress_bar flag)
   epoch_log_fields = dict(
       epoch=epoch,
       g_loss=current_loss_g,
       d_loss=current_loss_d,
       eta_seconds=eta_seconds,
   )
   log.info("epoch_end", **epoch_log_fields)
   ```

7. After the training loop ends (before the existing `# End of training` block), add:
   ```python
   # Emit training_complete (checkpoint_path and best_epoch populated in Plan 02;
   # emit with None values here to satisfy CORE-05 independently of QUAL-03)
   log.info(
       "training_complete",
       best_epoch=-1,
       best_val_metric=None,
       total_epochs=self.epochs,
       checkpoint_path=None,
   )
   ```

8. Keep existing checkpoint logic (`if checkpoint_dir and current_loss_g < best_loss`) UNCHANGED in this plan — Plan 02 replaces it. The existing save calls to `best_model.pt` / `last_model.pt` remain until Plan 02.

**Critical constraints (per CONTEXT.md):**
- Log events ALWAYS fire regardless of `progress_bar` value. Only `disable=not progress_bar` controls the tqdm bar.
- Do NOT guard structlog calls with `if progress_bar:`.
- `eta_seconds` is computed unconditionally; it is 0.0 only on the final epoch (correct).

```python
def fit(
```

Update the docstring to document the three new parameters:
- `progress_bar`: If True (default), display tqdm progress bar to stderr during training. Structured log events always emit regardless of this flag.
- `checkpoint_interval`: Save a validation checkpoint every N epochs. Default 10.
- `checkpoint_dir`: Optional directory to save best_checkpoint/ and final_checkpoint/ during training.

Update the `self.orchestrator.fit_all(...)` call to forward the new parameters:
```python
self.orchestrator.fit_all(

    progress_bar=progress_bar,
    checkpoint_interval=checkpoint_interval,
    checkpoint_dir=checkpoint_dir,
    **model_kwargs
)
```
